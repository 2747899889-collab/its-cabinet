# 「它的柜子」V1.0 App Store 上架准备说明

> 版本定位：V1.0 是本地 SwiftData 存储版本，不包含账号、云同步、扫码、图片上传、多宠档案、内购、广告和第三方统计。上架材料必须按实际功能描述，不能提前承诺未完成能力。

## 1. 当前版本可如实描述的功能

- 记录宠物食品/用品名称、生产日期、保质期、过期日期、数量和备注。
- 按「已过期、临期预警、安全期」排序展示柜子物品。
- 支持本地搜索已录入物品。
- 支持通用动态提醒规则和本地规则库查看。
- 支持体外驱虫、体内驱虫、疫苗三类健康周期提醒。
- 支持 iOS 本地通知授权、食品临期/过期通知、健康周期通知。
- 数据保存在用户设备本地。

## 2. 当前版本不能写进商店文案的能力

- 不能写「云同步」「换机自动恢复」「多设备互通」。
- 不能写「Apple ID 登录」「账号体系」。
- 不能写「扫码识别商品」「自动读取商品信息」。
- 不能写「图片上传」「拍照识别」。
- 不能写「多宠档案」。
- 不能写「绝不丢失数据」。V1.0 本地数据随设备和系统备份状态变化。

## 3. App Store Connect 信息建议

### App 名称

- 中文：它的柜子
- 英文备用：Its Cabinet

### 副标题

宠物食品过期与健康提醒

### 简短描述

「它的柜子」用于记录宠物食品和用品的保质期，并提醒疫苗、体内驱虫、体外驱虫等周期事项。V1.0 数据保存在本机，适合先验证核心流程。

### 关键词

宠物, 猫, 狗, 食品, 保质期, 过期提醒, 驱虫, 疫苗, 库存, 本地提醒

## 4. 隐私和权限

### 当前数据处理口径

- 本版本不创建账号。
- 本版本不上传用户数据到服务器。
- 本版本不接入第三方 SDK。
- 本版本不采集定位、联系人、相册、相机、广告标识符或支付信息。
- 本版本使用本地通知权限，仅用于提醒用户已录入的过期和健康周期事项。

### 隐私政策

即使 V1.0 不上传数据，也建议准备一份中文隐私政策页面，说明：

- App 保存哪些本地数据。
- App 为什么请求通知权限。
- 当前不提供云同步。
- 删除 App 可能导致本地数据被删除。
- 后续如果加入云同步、账号、扫码 API 或图片上传，会更新隐私政策并重新说明数据用途。

### App 隐私详情

提交前需要在 App Store Connect 填写 App 隐私详情。以当前 V1.0 功能看，口径应按「不收集数据」方向准备；如果后续加入云端账号、扫码 API、图片上传或统计 SDK，必须重新评估并更新。

## 5. 审核注意事项

- 商店截图必须来自当前真实功能，不展示云同步、扫码或账号页。
- 首次启动不要强制通知授权；当前设计是在「我的」页由用户主动开启通知，风险较低。
- 通知文案要具体、克制，只提醒用户自己录入的事项。
- 不要在审核备注里描述未完成的后续规划。
- 如果审核员问数据是否上传，回答：V1.0 数据仅保存在本机，不提供云端同步。
